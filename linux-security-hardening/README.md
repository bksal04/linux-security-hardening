# Linux Security Hardening

## Overview
This project demonstrates baseline Linux system hardening for a lab or server environment. It focuses on SSH security, firewall configuration, service reduction, audit checks, and basic brute-force protection.

## Objectives
- Reduce attack surface on a Linux host
- Harden remote administration
- Apply a host firewall
- Document service and account security checks
- Produce evidence that a system was reviewed and improved

## Controls Covered
- SSH hardening recommendations
- UFW firewall rules
- Fail2Ban deployment guide
- Basic audit script for common security checks
- Documentation of hardening decisions and validation steps

## Repository Contents
- `scripts/` contains a local audit script
- `configs/` contains example hardening configuration snippets
- `docs/` contains methodology, results, and remediation notes

## Recommended Evidence to Add
- Before-and-after screenshots of SSH configuration
- UFW status output
- Fail2Ban status output
- Audit script results from your own test machine

## Outcome
This project demonstrates a structured approach to Linux hardening rather than only listing security concepts.
